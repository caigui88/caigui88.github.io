Docker

学习路线：

[单机版Docker]概述——安装——命令——镜像——容器数据卷——DockerFile——Docker网络原理——IDEA整合Docker

[进阶版]Docker Compose——Docker Swarm（简化版的jenkins）

概述

开发和上线使用的是两套环境，应用环境，应用配置

开发 —— 运维；在我的电脑上正常，当迁移后出现了问题。

常见命令

  1. docker pull：拉取image
  2. docker images：查看本地image
  3. docker rm -rf：强制删除容器
  4. docker rmi：删除本地image
  5. docker build：根据dockerfile构建image
  6. docker save：将本地image以压缩包的形式保存下来，再通过docker load加载到本地
  7. docker run：创建一个image的实例——容器container
  8. docker stop：停止一个容器container
  9. docker statr：重新开启处于停止状态的容器container

全部指令官方文档：Docker 文档

创建一个数据卷：

  1. 数据卷的创建需要在容器创建时进行挂载，一旦容器创建好就不能再进行挂载。
  2. 数据卷可以帮助我们在主机环境去管理容器当中的环境与内容，由于每一个容器都是只包括容器所需要的最低限度的运行程序和脚本，因此可能在容器中很多指令无法应用，进而难以对容器的配置或文件进行修改等操作，通过挂载数据卷，相当于多了一个操作接口，进而可以在主机环境中对容器内的文件或配置进行修改。

Dockfile语法

镜像分层：

  1. Entrypoint入口：镜像运行入口，一般是程序启动的脚本和参数
  2. Layer 层：添加安装包、依赖、配置等，每次操作都形成新的一层
  3. BaseImage基础镜像：一些应用程序运行时所依赖的系统函数库、环境、配置和文件等

语法指令：

详细语法：官方文档

from：指定基础镜像

env：设置环境变量，可以在后面指令使用

copy：拷贝本地文件到镜像的指定目录

run：执行linux的shell命令，一般是安装过程的命令

expose：指定容器运行时监听的端口，给镜像使用者看，相当于一个端口使用说明

entrypoint：镜像中应用的启动命令，容器运行时调用

自定义镜像

容器网络

docker network create network_name

docker network connect network_name container_name

docker run -d --name new_container --network network_name -p port_number image_name

Docker挂载数据库

docker run --name mysql_container \
    -v /path/to/mysql-data:/var/lib/mysql \
    -v /path/to/my.cnf:/etc/mysql/my.cnf \
    -v /path/to/init.sql:/docker-entrypoint-initdb.d/init.sql \
    -e MYSQL_ROOT_PASSWORD=rootpassword \
    -p 3306:3306 \
    -d mysql:latest
