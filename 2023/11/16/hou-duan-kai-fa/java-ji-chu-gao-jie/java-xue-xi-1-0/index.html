修饰符

访问修饰符

  • default (即默认，什么也不写）: 在同一包内可见，不使用任何修饰符。使用对象：类、接口、变量、方法。
  • private : 在同一类内可见。使用对象：变量、方法。 注意：不能修饰类（外部类）
  • public : 对所有类可见。使用对象：类、接口、变量、方法
  • protected : 对同一包内的类和所有子类可见。使用对象：变量、方法。 注意：不能修饰类（外部类）。

非访问修饰符

  • static：用来修饰类方法和类变量。

    • 静态变量：

      static 关键字用来声明独立于对象的静态变量，无论一个类实例化多少对象，它的静态变量只有一份拷贝。 静态变量也被称为类变量。局部变量不能被声明为 static 变量。静态变量一定得进行初始化

    • 静态方法：

      static 关键字用来声明独立于对象的静态方法。静态方法不能使用类的非静态变量。静态方法从参数列表得到数据，然后计算这些数据。

    • 类内的静态变量和静态方法可以通过：

      类名.变量名/方法名 ClassName.staticValue/staticFunction

      进行调用。

  • final：用来修饰类、方法和变量，final 修饰的类不能够被继承，修饰的方法不能被继承类重新定义，修饰的变量为常量，是不可修改的。

  • abstract：用来创建抽象类和抽象方法。

    • 抽象类不能用来实例化对象，声明抽象类的唯一目的是为了将来对该类进行扩充。
    • 一个类不能同时被 abstract 和 final 修饰。如果一个类包含抽象方法，那么该类一定要声明为抽象类，否则将出现编译错误。
    • 抽象类可以包含抽象方法和非抽象方法。

  • synchronized：主要用于线程的编程。

    • synchronized 关键字声明的方法同一时间只能被一个线程访问。synchronized 修饰符可以同时配合其他四个访问修饰符一起使用。

  • transient 修饰符

    • 序列化的对象包含被 transient 修饰的实例变量时，java 虚拟机(JVM)跳过该特定的变量。
    • 该修饰符包含在定义变量的语句中，用来预处理类和变量的数据类型。

  • volatile：主要用于线程的编程。

String类

与c++不同，字符串在java中被抽象为一种引用对象，创建对象时，

可以使用直接创建的方式：

String str = "xxx"

也可以使用类实例化的方式：

String str = new String("xxx");

  1. 直接使用双引号“”创建的字符串会存储在字符串常量池中。

    1. 字符串常量池是Java专门用来存储字符串字面量的区域。它的一个特点是每当用直接创建的方式新创建一个字符串时，会先串池中查找是否存在相同内容的字符串，如果已存在，则直接返回串池内已存在的变量的引用，而不会创建新的对象。

  2. new关键字创建的字符串对象存储在堆区当中，每一个创建出来的变量都是独立的对象，在堆区里都有自己的地址，不共享。

字符串的“==”比较，比较的其实是字符串的引用，而不是字符串的实际变量值

String s1 = "Runoob";              // String 直接创建
String s2 = "Runoob";              // String 直接创建
String s3 = s1;                    // 相同引用
String s4 = new String("Runoob");   // String 对象创建
String s5 = new String("Runoob");   // String 对象创建

继承中成员方法的特点

  1. 就近原则:假设不使用super,直接调用函数名,因为存在隐含的this关键字,则是直接调用本类的方法,当在本类没找到时,则会向上查找父类是否存在.
  2. 方法重写:子类重写父类方法,因父类的方法不满足子类的需求,则需要对父类已经存在的方法进行重写.
    • 方法重写的原理:
      1. 子类从父类继承而来的虚方法表当中的method()会被子类中重写的method()进行覆盖,则虚方法表内的方法就被重写了.
    • 重写注意事项:
      1. 重写的方法名,形参列表必须和父类保持一致
      2. 重写的方法,访问权限子类必须大于父类( 空着不写 < protected < public )
      3. 子类重写父类方法时,返回值类型子类必须小于等于父类
      4. 重写的方法尽量与父类完全一致
      5. 只有在虚方法表当中的方法才可以被重写:
        1. 私有方法不可被重写;
        2. 子类不能重写父类的静态方法
@Override 重写注解
1. 放在重写的方法上,校验子类重写时语法是否正确
2. 加上注释,假如出现错误,则会有红色波浪线,表示存在语法错误
3. 加上后代码安全且优雅

多线程&JUC

一个进程当中可以有许多条线程，同时执行多种命令

充分利用程序当中的等待时间，充分利用硬件资源

基本概念

进程和线程

  1. 进程：一个应用程序or一个软件
  2. 线程：一个进程当中的某个指令、执行单元or执行场景。
  3. 注：一个进程可以有多个线程，也可以同时启动多个线程

对于Java这门语言来说，当执行一个class文件时，即是首先启动了JVM这个进程，而后JVM再启动==主线程main方法==；同时启动一个==垃圾回收线程==负责看护、回收垃圾。

多线程的目的

为了提高程序的处理效率，通过合理的资源分配方式，调度多项任务相间同时进行，充分利用硬件资源

线程的两种方式

  1. 并发：同一时间，多个指令在单个CPU上交替进行

  2. 并行：同一时间，多个指令在多个CPU上同时进行

线程对象具有生命周期

  • 新建状态**(NEW)**

    • 使用new关键字和Thread类或子类建立一个线程对象A后，则线程对象A处于新建状态。一直到进程start()这个线程。

  • 就绪状态**(RUNNABLE)**

    • 有执行资格，没有执行权
    • 当线程对象A调用start()方法后，线程进入就绪状态、参与到等待队列中，等待JVM中线程调度器的调度的调度。

  • 运行状态

    • 有执行资格也有执行权
    • 如果处于就绪状态的线程获取到了CPU资源，则执行run()方法，此时线程处于运行状态。处于运行状态的线程较为复杂，它有三种变化情况：阻塞、就绪与死亡状态。

  • 阻塞状态**(BLOCKED)**

    • 无执行资格也无执行权
    • 一个线程执行了sleep() （睡眠）、suspend()（挂起）等方法后，失去了当前抢占的资源，进入阻塞状态，当睡眠时间或阻塞条件结束后，重新进入就绪状态，再次参与CPU资源的抢夺，在抢夺成功后才可以继续执行阻塞方法下面的其他代码。
    • 阻塞分为：
      • 等待阻塞**(WAITING)**：运行状态中的线程执行了wait()方法；
      • 同步阻塞：线程在获取synchronized同步锁失败（因同步锁被其他线程占用；
      • 其他阻塞：通过调用线程的sleep() **(TIMED_WAITING)**或 join()发出了I/O请求时，线程进入阻塞状态。当sleep()状态超时，join()等待下次终止或超时，或者I/O处理完毕，线程重新进入就绪状态。

  • 死亡状态**(TERMINATED)**

    • 线程死亡，变成垃圾，被垃圾回收机回收
    • 一个运行状态的线程完成任务或者其他终止条件发生时，该线程就切换到死亡 状态。

Thread State

在JAVA当中，并没有规定运行状态，因为当线程抢夺到CPU执行权后，程序将由 JVM 交还给操作系统，因而不需要额外定义一个“运行”的状态。

  1. NEW
  2. RUNNABLE
  3. BLOCKED
  4. WAITING
  5. TIMED WAITING
  6. TERMINATED

JAVA中三种线程实现方式

  1. 继承java.lang.Thread类，重写run方法（线程和任务绑死，无返回值）
  2. 实现java.lang.Runnable接口 ，实现run方法，实例化子类对象，传入Thread当中（线程和任务进行分离，无返回值）
  3. 使用Callable接口和Future接口方式实现（线程和任务进行分离，有返回值）

线程调度

抢占式：哪个线程优先级高，抢到的CPU时间片的概率就高一些/久一些（JAVA采用抢占式调度模式）

均分式：平均分配CPU时间片，每个线程占用的CPU时间片时间长度一样（主打一个众生平等）

线程优先级

JVM自己创建的线程main 的优先级为5

守护线程

当其他非守护线程结束以后，守护线程才会陆续结束。即守护线程一定在非守护线程结束以后才结束。

应用场景

  1. 聊天（线程1）与输入文件（线程2），在线程1结束后线程2没有存在必要，将线程2设置成守护线程，在线程1结束后紧接着也结束。

还需要一个等待平台去唤醒其他线程。

理想情况

生产者优先于消费者抢夺到CPU资源，成功产出数据后；消费者抢夺到CPU资源并且消费生产者产出的数据。

实际情况

  1. 消费者等待：消费者每次抢夺到CPU资源时，会先判断是否存在数据，若无数据则进入wait操作，不参与CPU资源的抢夺，生产者成功抢夺CPU资源；在生产者产出数据后，还需要去唤醒处于等待状态的消费者。；若有数据则消耗数据。
  2. 生产者等待：生产者每次抢夺到CPU资源时首先进行判断，假设判断已经产出数据，则进入wait操作，不参与CPU资源的抢夺，消费者成功抢夺到CPU资源，在消耗资源后，消费者再次抢夺到资源，但是检测到无数据，则唤醒处于等待的生产者，并且自身进入等待。

线程池

  1. 创建一个线程池，池子内是空的
  2. 提交任务，池子会创建新的线程对象，任务执行完毕将线程归还线程池；下次提交任务时，不需要额外创建新的线程，直接复用已有线程
  3. 如果提交任务时，池子内没有空闲线程且无法创建新的线程，任务排队等待

网络编程的三要素

  1. IP：设备在网络中的地址，是设备的唯一标识
  2. 端口号：应用程序在设备中唯一的标识
  3. 协议：数据在网络中传输的规则，常见的协议有UDP、TCP、http、https、ftp

IP(Internet Protocol)

互联网协议地址，是分配给上网设备的数字标签

  1. IPv4

    1. 采取32位地址长度，分为四组，每组八位，如：$11000000 10101000 00000001 01000010$
    2. 为了便于使用和记忆，采用点分十进制表示法：$1100000010101000 00000001 ~01000010\rightarrow192.168.1 66$

    弊端：数量较少，不够更多设备使用

  2. IPv6

    1. 采取128位地址长度，分成八组，每组16位
    2. 采用冒分十六进制表示法，省略高位的0

  3. 特殊的IP，本机IP&回收地址&本地回环地址。$127.0.0.1$ localhost。永远只会寻找当前所在本机
