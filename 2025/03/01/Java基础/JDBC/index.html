JDBC

通过I/O操作对程序运行过程中出现的数据进行持久化，但保存下来的数据并不具备结构和逻辑，不便于管理与维护

通过关系型数据库，将数据进行存储，交由数据库进行维护，

  • 关系型数据库是通过库和表分隔不同的数据，表中数据存储的方式是行列，区分相同格式的不同数据。

基础

JDBC

概念：

JAVA Database Connectivity Java的数据库连接

JDBC为JAVA的一组独立于任何数据库管理系统的API

JDBC只提供接口规范，由各个数据库厂商提供实现，使用jar包，导入jar进行使用

体会接口编程，只关心接口和规范，无需关心实现

不同的数据库使用不同的驱动jar包

對JDBC的使用

Driver注冊驅動

  • Class.forName("com.sql.cj.jdbc.Driver");
  • 儅使用JDBC鏈接數據庫時，需要加載數據庫特定的驅動程序，以便與數據庫進行通信，加載驅動程序的目的是爲了注冊驅動程序，使得JDBC API能夠識別并與特定的數據庫進行交互。
  • 從JDK6開始，無需顯式調用Class.forName()來加載JDBC驅動程序，只要在類路徑中集成了對應的jar包，會自動在初始時完成驅動程序的注冊。

Connection

  • 建立鏈接需要指定三個參數：數據庫url、用戶名、密碼

    • URL：“jdbc:mysql://localhost:3306/atguigu”

      jdbc:mysql//IP地址：端口號/數據庫名稱?參數鍵值對1&參數鍵值對2

  • Connection接口還負責管理事務，Connection接口提供了commit和rollback方法，用戶提交事務和回滾事務。

  • 可以創建Statement對象，用於執行SQL語句并與數據庫進行交互。

  • 在使用JDBC技術時，必須要先獲取Connection對象，在使用完畢后，要釋放資源，避免資源浪費以及泄露。

Statement

用於執行SQL語句並於數據庫進行交互

通過Statement對象，可以向數據庫發送SQL語句並獲取執行結果

  • 結果可以是一個或多個

    • 增刪改：受影響行數單個結果
    • 查詢：單行單列，多行多列，單行多列

  • 使用Statement會產生SQL注入的問題，即通過使用如‘’對SQL原來的結構進行破壞，使得SQL語句變成永真句，進而完成注入，非法訪問得到數據。

PreparedStatement

本身為Statement的子接口，繼承于Statement接口，用於解決StatementSQL注入的問題。

解決SQL注入的方法為：

  • 對傳入的SQL語句進行預編譯，使用一對‘’將用戶輸入的sql包裹起來，儅用戶輸入的語句含有‘’時，會將輸入的語句中的‘’使用轉義字符\‘和\’代替，以防止SQL注入的攻擊

ResultSet

  • ResultSet.next()首先判斷是否存在下一行，若有則游標向下移動一行；若無則停止。

  • 然後使用ResultSet.getXXX()方法來根據列名讀取相應的數據。

进阶

JDBC拓展

实体类和ORM

一个表对应一种对象

表中的一列数据对应对象的一种属性

因为数据库中的数据是零散的，通过将数据库中的数据对应到Java当中的一个对象上，以对象作为载体进行数据的操作，这个载体就是实体类

ORM-Object Relational Mapping

对象到关系数据库的映射：在编程中，将面向对象的概念和数据库中表的概念对应起来，以面向对象的角度操作数据库中的数据——一张表对应一个类；一行数据对应一个实例；一列对应一个类的属性

修改刚刚注册的用户数据。

批量操作

对于一次需要插入多个值，如果只是单纯采用for-i循环进行插入，由于需要多次进行建立连接的建立和断开，性能差，时间开销大：

因此采取批量操作：

通过将多个值合并入同一条待执行的sql语句，则只需要建立一次连接即可完成值的插入，大大提高了效率

连接池

池技术的核心思想：通过对已创建资源的复用，避免不必要的重复创建和销毁。

预先创建一部分Connection对象，通过配置，由连接池负责创建连接，管理连接，释放连接等操作

常见连接池

  • DBCP
  • C3P0
  • Proxool
  • Druid 德鲁伊
  • Hikari(shi ga li)

JDBC优化以及工具类封装

在用到了进阶篇的内容后，发现仍存在代码冗余的问题：

  1. 一个项目中存在多个连接池；
  2. 获取链接
  3. 链接的回收

使用ThreadLocal进行优化

通过ThreadLocal让每个线程只使用一个连接进行数据库的操作，避免反复获取和回收连接导致的资源浪费。

效果展示

可以看到，如果使用Util版本的连接池，则会获取到三个不同的连接

而使用了Util2的连接池后，则一个线程里只会获取到一个链接，并且进行反复使用

DAO封装与BaseDAO工具类

DAO Data Access Object 数据访问对象

让数据在JAVA中以对象的形式存在，做到：

一张表对应一个实体类，一张表的操作对应一个DAO对象

在JAVA操作数据库时，我们将对同一张表的增删改查作统一的维护，维护的类就属于DAO层面

DAO只关注数据库的操作，供业务层Service调用
