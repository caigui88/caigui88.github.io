01. 数据结构与算法 | 算法通关手册（LeetCode）

Introduction

对于数据结构而言，可以从逻辑上的一对一、一对多、多对多进行分类，其中线性结构是一对一的经典数据结构，而隶属于线性结构比较常用的就有 Array「数组」和 LinkedList「链表」，以及由此加上一定数据或操作约束的栈、队列、哈希表等等。

本文结合网上的资料进行线性数据结构的总结与复习

Array

简介

数组的定义

一种，使用一组连续的内存空间来存储一组具有相同类型数据的，线性数据结构。

从基本定义当中提炼的要点有：

  • 连续的内存空间
  • 数据的类型相同

也是因为上面这两点原因才可以让数组以“首地址➕索引值*类型大小”的方式快速访问数组内的任意数组，也就是随机访问元素。

多维数组

如果只有数组的分布只是一行，那么就是一维的，而后可以根据需求增加数组的维度，例如处理矩阵问题时可以采用二维的数组，这在直观想象中就是一个 m*n 的矩阵，有行有列，而更高维度的其实就是需要去对每一个数组进行限定，确定每一个索引的意义。

基本操作

访问元素

时间复杂度 O(1)，原因是可以根据索引直接访问到数据

查找元素

查找特定元素需要 O(n) 的复杂度，原因是需要对数组进行遍历，逐个比较来查找「也就是线性查找」。

插入元素

在尾部插入元素是 O(1)

在数组头部和中间的插入都是 O(n)

修改元素

时间复杂度 O(1)，修改操作在线性结构中只涉及到“查”操作，因此当“查”是 O(1) 时，那么“改”也是一样的。

删除元素

删除尾部

时间复杂度是 O(1)

删除指定位置元素

时间复杂度是 O(n)

基于条件删除元素

时间复杂度是 O(n)
